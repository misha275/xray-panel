основные таблицы:
1. [ip адрес, порт, ссылка-подписка с конфигом]
2. [таблица пользователей]

есть множество серверов, на каждом из которых настроен по шаблону инбаунд с 1 клиентом. конфиг клиента отправляется на главный сервер где хранится в таблице [1]. на главном в loop{} раз в несколько минут проверяется нет ли новых серверов в таблице (дополнительно можно проверять активны ли все сервера) и если появляется новый сервер, создается новый инбаунд на главном с маршрутом *новый инбаунд -> новый сервер*. для каждого клиента из таблицы пользователей [2] постоянно проверяется сужествует ли его клиент на всех доступных серверах, если не существует, создается, если есть на несуществующем сервере, конфиг удаляется из массива конфигов.
